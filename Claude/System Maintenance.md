# System Maintenance

*Keeping the Second Brain healthy and functional*

## Weekly Maintenance (Sundays)

### 1. Create Weekly Log
Create `Claude/Weekly Logs/Week of YYYY-MM-DD.md` from daily notes in `Claude/Daily Notes/`

See [[Session & Activity Logging]] for full protocol details.

### 2. Update Task Management
Update [[Task Management/INDEX]] if needed:
- Check if priorities have shifted
- Update deadlines that have passed
- Reflect completed work

### 3. System Health Check (5 minutes)

**File Placement Audit:**
- Scan root for misplaced files
- Check daily notes for content needing permanent homes
- Move orphaned content to appropriate folders

**Link Integrity Check:**
- Test major navigation paths
- Verify new content is discoverable through links
- Identify "orphaned information" that exists but isn't connected
- Check INDEX.md files: Ensure they're up-to-date with current projects and docs

**Document in Weekly Log:**
- File drift status (clean / minor issues / needs attention)
- Link health (all paths working / some broken links)
- Action items for next week

### 4. Suggest Adjustments
Based on patterns observed, suggest improvements to system structure or workflows.

---

## When to Add Complexity

**Pattern Detection Rule:** Same pattern appears 3-4 times

**Add structure when:**
- Current approach creates friction
- User explicitly asks for more structure
- Navigation becomes genuinely difficult

**Don't add structure when:**
- It's just once or twice
- Current simple approach is working
- Complexity would be premature optimization

---

## Evolution Guidelines

**Design Principle:** System must grow linearly — information can expand infinitely, but access/update time stays constant.

**Key Practices:**
- One-line logs where possible
- Batch updates (weekly, not daily)
- Natural navigation through links
- Small networked files, not monoliths

**User's instinct for simplicity is right. Follow their lead, not your complexity bias.**
